Github Pages加速

2021-02-20

本网站是我使用Github Pages+Gridea搭建的静态博客，域名当初买了趁着搞活动41元买了10年，计划长期使用，然而使用过程中发现这套配置唯一的短板就是国内访问速度太慢，因此收藏几个可以加速访问的方法。

加速Github Pages:

  • Github Pages访问太慢？通过Netlify免费加速。
    https://www.shuzhiduo.com/A/LPdovX1O53/

免费，以前用过一段时间，速度还是不错的，但也有一些问题：
一是经常会出现点击登录之后就发生错误，导致无法登录，累觉不爱；其次用了一段时间后打开博客发现网速好慢，感觉效果可能不太行了。（后面我是在家里访问，也许是因为这边网络比较差。）


  • 优雅的使用Vercel加速GitHub Pages
    https://www.iaia.cc/posts/2418998582.html

    vercel是什么神仙网站？
    https://www.jianshu.com/p/9f67ff433d8e

免费，加速效果还是不错的，免费还要啥自行车。印象中之前Netlify配置好后速度和Vercel一样挺快。
前几次弄好后发现不但没有加速，反而博客打不开了。最后看了邮件，原来也一定要加入另外两条解析记录。
第二篇文章中更深入介绍了vercel的功能，它甚至能构建动态网站，但是好像要付费？


  • hexo+github+coding双线部署（加快博客访问速度）
    https://leflacon.top/b61b079a/

coding是国内的网站，可以同时部署博客在它上面，类似的好像还有码云等。这种方法原理大概和Github Pages一样，只是服务器在国内，不经过墙，所以速度快点。可以设置墙外访问Github Pages，墙内访问coding。
免费，但是考虑到要往上面上传文件、代码等，想想是国内网站还是算了😅（一是不信任，二是部署网站好像要实名认证/备案）。不过相比于其他几种方法，这种方法的好处应该在于它能够让博客被百度收录，详见如何让百度收录 GitHub Pages 个人博客。


  • 使用国内CDN加速
    https://www.cnblogs.com/geraint/p/13115459.html

国内CDN主要是腾讯云、阿里云等这些国内互联网公司提供的服务。
收费，不过腾讯云会送新人一些流量，够用好几个月了。但我不想&懒得备案域名，所以没用过。



2021.4.7补充：
今天发现了另一个CDN加速的网站CloudFlare，可以免费使用（如果免费的满足不了你，也可以付费）。
我用了下免费套餐，感觉速度比Vercel要更快点。Vercel有时候打开还是有点慢的，但是CloudFlare加速后都挺快的（以后不好说，但是最起码刚弄好是这样的）。

网站如下：
https://dash.cloudflare.com/
