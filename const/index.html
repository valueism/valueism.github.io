修饰符const

2021-02-15
  1. const为了防止变量被修改，因此必须初始化

const int i= get size（）；//正确：运行时初始化
const int 3=42； //正确：编译时初始化
const int k; //错误：k是一个未经初始化的常量

2.const原理是预处理，会在运行时把常量替换成数字。因此const对象只在文件内有效
我们想让这类 const对象像其他（非常量）对象一样工作，也就是说，只在一个文件定义 const，而在其他多个文件中声明并使用它。解决的办法是，对于 const变量不管是声明还是定义都添加 extern关键字，这样只需定义一次就可以了：(C++primer P54.)

//file1.cc定义并初始化了一个常量，该常量能被其他文件访问
extern const int BuffSize = fcn（）;
//file1.h头文件
extern const int bufsize；//与fi1e1.cc中定义的 BuffSize是同一个

fi1e1.h头文件中的声明也由 extern做了限定，其作用是指明 bufsize并非本文件所独有，它的定义将在别处出现。

3.指针和const
const和指针一起用时，容易搞不清到底是指向常量的指针，还是指针本身是常量。

来源：https://blog.csdn.net/ClaireSy/article/details/108422945?spm_id_from=333.788.b_636f6d6d656e74.41
//例一：const修饰的是指针，指针指向可以改，指针指向的值不可以更改
const int * p1 = &a;
p1 = &b; //正确
//*p1 = 100; 报错

//例二：const修饰的是常量，指针指向不可以改，指针指向的值可以更改
int * const p2 = &a;
//p2 = &b; //错误
*p2 = 100; //正确

//例三：const既修饰指针又修饰常量
const int * const p3 = &a;
//p3 = &b; //错误
//*p3 = 100; //错误

最好的方法是从右往左读，例一中离p1最近的是*，说明这是一个指针。左侧的const int表明这个指针指向一个常量的整型对象。因此，p1是一个指向整型常量的指针。

例二中离p2最近的是const，说明p2这个变量本身是常量，并且p2的变量类型由声明符确定。左侧的int*表明了p2的类型是整型指针。因此，p2是一个指向整型的常量指针。
